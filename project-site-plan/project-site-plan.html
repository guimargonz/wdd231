Site Plan: TherapyTrack Pro

1. Site Name

Name: TherapyTrack Pro

Reason: The name combines "Therapy" with "Track" to suggest management and progress monitoring, and "Pro" implies a professional tool for therapists. It's concise and relevant.

Optional Domain Idea: `therapytrackpro.app` or `speechpathpro.com`

2. Site Purpose

TherapyTrack Pro aims to be a user-friendly, web-based tool designed to help independent speech-language pathologists (SLPs) or small clinics efficiently manage patient information, track therapy session details, and monitor progress towards goals. The site will provide a secure (simulated for this project) dashboard for therapists to view patient lists, add new patients, log session notes, set therapy goals, and visualize (basic) progress. It will simplify administrative tasks, allowing therapists to focus more on patient care. The site will *not* handle real Protected Health Information (PHI) but will simulate the structure and functionality for such a system.

3. Scenarios

  • "How can I quickly see a list of all my active patients and their next scheduled (hypothetical) appointment times?"
  • "I just finished a session with a patient; where can I log my session notes, the activities we did, and their performance on specific goals?"
  • "How can I add a new patient to my caseload and set initial therapy goals for them?"
  • "Is there a way to view a patient's progress over the last few sessions for a particular speech sound or language target?"

#4A90E2

Secondary: Professional Grey
#7F8C8D

Accent: Encouraging Green
#2ECC71

#F8F9FA

Usage:

  • Calming Blue (#4A90E2): Main navigation, primary buttons, headers, key interface elements that require user attention or action.
  • Professional Grey (#7F8C8D): Subheadings, borders, secondary text, disabled states, subtle background elements.
  • Encouraging Green (#2ECC71): Success messages, progress indicators, "add new" or confirmation buttons.
  • Soft Off-White (#F8F9FA): Main content background for a clean, uncluttered look.
  • Deep Charcoal (#343A40): General body text, labels, and data display for optimal readability.

6. Wireframes (Example: Patient Dashboard/List Page - Home)

The following are conceptual wireframes for the main patient dashboard (home page) layout.

Mobile View Wireframe (Patient Dashboard):

Header: Site Name/Logo, Hamburger Menu.
Page Title: "My Patients".
Action Button: "+ Add New Patient".
Patient List: Vertically stacked cards. Each card shows: Patient Name, Next Appointment (placeholder), Brief status/goal. Clickable to view details.
Filter/Search (Optional): Small search bar or filter icon.

Desktop View Wireframe (Patient Dashboard):

Header: Site Name/Logo (left), Horizontal Navigation (e.g., Dashboard, Patients, Resources, Settings).
Main Content Area:

  • Left Sidebar (Optional for quick stats/calendar): Placeholder for quick stats or mini calendar.
  • Main Section:
    • Page Title: "Patient Dashboard".
    • Action Button: "+ Add New Patient".
    • Patient List Table: Columns for Patient Name, Age/DOB, Last Session, Next Session (placeholder), Primary Goal, Actions (View, Edit).

Note: Other pages would include a "Patient Detail Page" (viewing/editing specific patient info, goals, session logs) and an "Add/Edit Session Page" (form for logging session details).
